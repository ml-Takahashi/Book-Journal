# 読書ジャーナル（Book Journal）

読書体験を整理するためのアプリケーション群をまとめたリポジトリです。現在は React + TypeScript 構成のフロントエンドを中心に実装しており、将来的には API サーバーや永続化層などのバックエンドを追加できるようモノレポ構成を前提にしています。

## リポジトリ構成

- `book-journal-frontend/` Vite + React による UI 実装。ジャンルツリーのブラウズ、書籍カードギャラリー、詳細編集ダイアログなどを含む読書ジャーナルのフロントエンドです。
- `package-lock.json` ルートで依存関係のバージョンを固定するロックファイル（モノレポ化を見据えて維持）。

> 📌 フロントエンド内のより詳しい構成は `book-journal-frontend/README.md` を参照してください。

## フロントエンドのセットアップ

```bash
cd book-journal-frontend
npm install
npm run dev
```

ターミナルに表示される URL（既定では `http://localhost:5173`）へアクセスするとアプリを確認できます。HMR が有効なため、UI 更新が即座に反映されます。

## 将来的な拡張（バックエンド想定）

- 認証・ユーザー管理の導入（想定: OAuth もしくは認証サーバー連携）
- 書籍メタデータの永続化 API（想定: Node.js/Express + データベース）
- 同期・共有機能の実装（想定: REST/GraphQL エンドポイント）

バックエンドを追加する際は、`apps/` や `services/` などのディレクトリを新設し、それぞれに独立した README とパッケージ設定を追加することでモジュールごとの責務を明確に管理できます。
